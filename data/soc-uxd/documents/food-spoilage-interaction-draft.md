## 1. 设计摘要

食物腐败模块用于让可腐败食物在非冷藏状态下随服务器时间损耗，并通过烹饪、通电冰箱或特定冷藏环境延缓或暂停腐败。交互设计目标不是新增重型管理界面，而是在背包、容器、冰箱、烹饪、拾取和物品详情等既有界面中，低打扰地表达食物状态。

本次交互草稿优先覆盖 P0：可腐败状态展示、剩余保质期、即将腐败弱提示、冷藏生效/失效、腐败转化、烹饪重置保质期、同步刷新和异常提示。

### 1.1 历史经验依据

- 组件库：优先复用通用物品 tips、文本 tips、Toast、确认弹窗、状态按钮、背包格子、容器列表等通用组件。
- 容器系统：普通容器、死亡盒子、掉落物、PC 背包框架已有大量共用逻辑；本功能应作为物品状态扩展，不另建独立食品管理页。
- 制造系统：材料不足、未解锁、可制作等状态通过格子标签、详情说明、按钮状态表达；食物腐败可沿用“状态标签 + 详情解释 + Toast”的表达方式。
- 双端规则：PC 需要 hover、快捷键提示与横向布局适配；移动端以点击/长按查看详情为主，避免在格子内塞入过多文字。

### 1.2 设计边界

| 类型 | 本次交互负责 | 不在本次交互中展开 |
| --- | --- | --- |
| 信息表达 | 食物是否会腐败、多久腐败、是否冷藏、腐败后产物 | 食物品质等级、多阶段腐败链、食品科学模拟 |
| 页面范围 | 背包、物品详情、普通容器、冰箱、烹饪、拾取、Toast、可选确认弹窗、帮助图例 | 新食品管理页、复杂排序筛选系统、活动入口 |
| 反馈方式 | 弱提示、格子状态、详情文本、局部刷新 | 远距离强弹窗、战斗中强打断、复杂专属动画 |
| 系统规则 | 服务端权威、客户端展示、状态同步与刷新 | 服务端具体实现、数值平衡、经济产出最终值 |

## 4. 核心流程

### 4.1 查看食物状态

1. 玩家打开背包/容器/拾取列表。
2. 客户端请求或使用服务端同步的物品实例状态。
3. 可腐败食物显示剩余保质期或状态标识。
4. 不可腐败食物不显示腐败信息。
5. 玩家点击/长按/hover 食物时，物品详情展示完整状态。

### 4.2 普通腐败流程

1. 食物处于背包、普通容器或世界掉落物中。
2. 服务端按真实时间推进腐败。
3. 未到期时展示剩余保质期。
4. 低于阈值时进入“即将腐败”弱提醒状态。
5. 到期后转化为腐败产物、失效或销毁。
6. 客户端刷新图标、名称、详情和可使用状态。

### 4.3 冷藏生效/失效流程

1. 玩家将食物放入通电冰箱。
2. 服务端标记冷藏状态，记录冻结剩余时间。
3. 客户端显示冷藏图标、冷藏说明和暂停计时状态。
4. 冰箱断电或食物移出冷藏容器后，服务端取消冷藏状态。
5. 客户端刷新为继续腐败状态，并通过 Toast 或界面内提示说明。

### 4.4 烹饪重置保质期流程

1. 玩家把可腐败材料放入烹饪界面。
2. 材料格展示当前剩余保质期和是否即将腐败。
3. 产物预览显示完成后新食物保质期。
4. 烹饪完成时，服务端销毁原物品并生成新食物实例。
5. 新食物按配置获得完整保质期。
6. 若材料在结算时已腐败，烹饪失败或产出腐败结果，并 Toast 提示。

### 4.5 同步冲突流程

1. 客户端本地显示与服务端状态不一致。
2. 玩家查看、移动或使用食物时触发服务端校验。
3. 服务端返回最新状态。
4. 客户端局部刷新格子/详情。
5. 必要时显示 `物品状态已变化` 或 `物品状态已同步`。

## 5. 页面与组件规格

### UI-001 背包界面 - 食物格子腐败状态

**定位：**已有背包页面改造，不新增入口。

**推荐组件：**既有背包格子组件 + 状态标签/角标 + 可选进度条 + 物品 tips。

**格子状态优先级：**

1. 同步刷新中：格子弱禁用或局部刷新态，暂不允许拖拽/使用。
2. 已腐败产物：显示腐败产物图标与名称，不再保留原食物名称。
3. 冷藏中：显示冷藏图标，剩余时间保持冻结值。
4. 即将腐败：显示弱警示标识或警示色时间。
5. 正常可腐败：显示普通剩余时间。
6. 不可腐败：不显示腐败信息。

**交互规则：**

- 移动端：点击选中，长按或点击详情入口查看完整信息。
- PC：hover 展示物品 tips；点击选中；拖拽移动后等待服务端确认。
- 拖拽期间如果服务端判定已腐败，拖拽失败并刷新为腐败产物。

**程序注意：**格子展示不能以客户端本地计算为最终状态；打开背包、移动物品、断线重连时需要刷新。

**GUI注意：**格子内信息空间有限，冷藏/即将腐败/已腐败的视觉优先级高于完整时间文本。

### UI-002 物品信息面板 - 食物腐败详情

**定位：**食物腐败信息的主解释入口。

**推荐组件：**`UiItem Tips` / 项目既有物品详情面板。

**信息结构：**

- 食物名称、图标、类型、食用效果。
- 保质状态：`剩余保质期：{time}`。
- 当前状态：正常腐败中 / 冷藏中 / 即将腐败 / 已腐败 / 不会腐败。
- 腐败后产物：`腐败后产物：{itemName}`。
- 说明文本：根据状态切换。
- 操作按钮：使用、拆分、丢弃等沿用既有物品操作。

**状态规则：**

| 状态 | 详情面板表现 |
| --- | --- |
| 可腐败正常 | 显示剩余保质期和腐败后产物。 |
| 即将腐败 | 剩余时间使用弱警示表现，显示 `即将腐败`。 |
| 冷藏中 | 显示冷藏图标和 `冷藏中，腐败倒计时暂停`。 |
| 不可腐败 | 可显示 `不会腐败`，也可隐藏整块保质信息，需 GUI/交互确认。 |
| 已腐败 | 面板切换为腐败产物详情，显示使用限制。 |

**待确认：**不可腐败食物是否需要在详情里显示 `不会腐败`。从低 UI 噪音原则看，建议默认隐藏，仅在帮助图例里解释。

### UI-003 普通储物箱/容器界面

**定位：**已有容器页面改造，强调普通容器不阻止腐败。

**历史依据：**容器系统已有 PC 背包/容器框架，多个容器可以复用同一套格子和拖拽逻辑。

**交互规则：**

- 打开容器时刷新容器内食物状态。
- 食物在普通容器内继续腐败。
- 首次打开普通容器且容器中有可腐败食物时，可显示弱说明 `普通容器不会阻止食物腐败`。
- 后续不常驻强提醒，避免增加仓库整理压力。

**拼接注意：**普通容器、尸体、掉落物、战利品箱的拖拽逻辑可能不同，需要在接入时逐类确认。

### UI-004 冰箱/冷藏容器界面

**定位：**本功能核心差异界面。

**状态：**通电冷藏中、断电失效、电力不足、冷藏恢复、容器为空、食物正常、即将腐败、已腐败、同步刷新中。

**界面区块：**

- 容器标题：冰箱名称。
- 电力状态：通电 / 断电 / 电力不足。
- 冷藏状态：冷藏生效 / 冷藏失效。
- 冷藏说明：通电时说明暂停腐败，断电时说明将继续腐败。
- 食物格子：显示冷藏图标和剩余保质期。
- 玩家背包区：沿用容器/背包交互。

**断电提示规则：**

- 玩家主动打开断电冰箱时，界面内展示常驻断电提示。
- 玩家正在查看冰箱时发生断电，显示 Toast + 界面内状态刷新。
- 玩家远离冰箱时不强弹窗。

**程序注意：**冰箱通断电、容器销毁、食物移入/移出都必须触发服务端重新判定冷藏状态。

### UI-005 烹饪界面 - 材料与产物保质期

**定位：**已有烹饪页面改造，表达“烹饪生成新食物并重置保质期”。

**推荐表现：**

- 材料格显示材料当前剩余保质期。
- 材料即将腐败时，材料格出现弱警示。
- 产物预览显示产物保质期，例如 `熟肉 24h`。
- 规则说明可在首次提示或 info tips 中展示：`烹饪完成后按产物配置重置保质期`。

**按钮状态：**

- 材料正常：开始烹饪可用。
- 材料已腐败：开始烹饪禁用或点击后 Toast `材料已腐败，无法完成烹饪`。
- 烹饪中材料腐败：结算时服务端校验失败，刷新材料/产物状态。

**动效注意：**不需要专属腐败动画；烹饪完成可沿用既有完成反馈，新增产物保质期展示即可。

### UI-006 拾取/战利品列表界面

**定位：**P1，避免玩家拾取后才发现食物已腐败。

**规则：**

- 列表项展示剩余腐败时间、即将腐败、已腐败、不可腐败。
- `全部拾取` 时，如果有状态变化，拾取完成后使用 Toast 合并提示。
- 若拾取过程中食物腐败，以服务端结果为准，刷新列表或背包。

### UI-007 Toast 弱提示

**推荐复用：**通用 Toast。

**提示合并规则：**

- 同一批次多个食物腐败，合并为 `部分食物已腐败`。
- 连续移动多个食物进入冷藏，只提示一次 `食物已进入冷藏状态`。
- 连续点击不可食用腐败产物，3 秒内只提示一次。

### UI-008 冰箱断电提示

**推荐形式：**界面内提示优先，Toast 辅助；不建议远距离强弹窗。

**按钮：**如使用弹窗或提示面板，仅保留 `知道了`，不额外做跳转，除非电力系统已有可复用跳转入口。

### UI-009 腐败产物详情

**规则：**

- 显示腐败产物名称、来源、使用限制、材料用途、处理方式。
- MVP 若不可食用，则使用按钮禁用或点击 Toast `该物品无法食用`。
- 若后续允许食用且有负面效果，再启用 UI-011 二次确认弹窗。

### UI-010 服务器同步刷新表现

**触发：**登录、重连、打开容器、移动食物、腐败转化、冰箱通断电。

**表现：**

- 单格刷新态优先，不遮挡整个背包。
- 刷新中格子可弱禁用拖拽/使用。
- 同步完成后如状态变化，Toast `物品状态已变化` 或 `物品状态已同步`。

### UI-011 腐败产物食用确认弹窗

**启用条件：**仅当腐败产物配置为可食用且存在负面效果时启用。当前 MVP 推荐不启用。

**推荐组件：**`ComModal` 通用确认弹窗。

**按钮：**取消 / 仍然食用。

### UI-012 食物状态图例说明

**入口建议：**

- 首次查看可腐败食物时用轻量引导气泡。
- 点击冷藏/腐败状态图标时打开文本 tips。
- 设置或帮助中保留状态说明。

**待确认：**是否需要 `不再提示`。若已有新手引导系统，建议沿用既有“不再提示”逻辑。

## 6. 状态矩阵

### 6.1 食物格子状态矩阵

| 状态 | 显示 | 可操作 | 触发条件 | 退出条件 |
| --- | --- | --- | --- | --- |
| 不可腐败 | 普通食物格子，无腐败信息 | 正常 | `CanSpoil=false` | 无 |
| 可腐败正常 | 剩余时间，如 `06h` | 正常 | 剩余时间高于阈值 | 进入冷藏、低于阈值、腐败 |
| 即将腐败 | 弱警示 + 短时间，如 `!02h` | 正常 | 剩余时间低于 `SpoilWarningThreshold` | 食用、烹饪、冷藏、腐败 |
| 冷藏中 | 冷藏图标 + 冻结剩余时间 | 正常 | 通电冰箱/冷藏环境 | 离开冷藏、断电、环境失效 |
| 已腐败 | 腐败产物图标/名称 | 依产物配置 | 到期并完成转化 | 丢弃、使用、回收 |
| 同步刷新中 | 刷新态/弱禁用 | 暂不可拖拽或使用 | 等待服务端状态确认 | 同步完成 |
| 数据异常修复 | 局部刷新，不额外强提示 | 暂不可操作 | 服务端修复异常数据 | 刷新为最终状态 |

| 状态 | 必显信息 | 可选信息 |
| --- | --- | --- |
| 可腐败正常 | 剩余保质期、腐败后产物 | 非冷藏说明 |
| 即将腐败 | 剩余保质期、即将腐败 | 处理建议，如烹饪/冷藏 |
| 冷藏中 | 冷藏中、腐败倒计时暂停、剩余保质期 | 冷藏来源 |
| 冷藏失效 | 未冷藏、将继续腐败 | 断电说明 |
| 已腐败 | 腐败产物说明、是否可食用 | 材料/回收用途 |
| 不可腐败 | 普通食物详情 | `不会腐败` 图例说明 |

### 6.3 冰箱界面状态矩阵

| 状态 | 顶部信息 | 食物格子 | 反馈 |
| --- | --- | --- | --- |

| 断电 | 断电警示、冷藏失效 | 去掉冷藏图标，恢复倒计时 | Toast + 界面内提示 |

| 容器为空 | 空格子 | 无食物状态 | 不提示 |

## 7. 双端交互规则

### 7.1 移动端

- 格子内只保留最关键状态：冷藏、即将腐败、已腐败、短剩余时间。
- 完整说明放到物品详情或状态图例中。
- 使用点击/长按查看详情，不依赖 hover。
- Toast 显示 2-3 秒，不打断战斗、探索、建造。

### 7.2 PC 端

- 沿用 PC 背包/容器框架，食物腐败作为格子状态扩展。
- hover 物品格子时显示完整物品 tips。
- 拖拽、快捷拾取、全部拾取、整理等既有快捷键逻辑保持不变。
- 同步刷新中，单格弱禁用，避免拖拽复制/丢失。
- `KeyPromptBar` 使用框架层，不在单个功能界面里重复制作。

### 7.3 PC 是否需要独立组件

| 模块 | 判断 |
| --- | --- |
| 背包格子 | 复用既有 PC 背包格子，新增状态层即可。 |
| 物品 tips | 复用既有 tips，增加保质状态信息块。 |
| 冰箱状态栏 | 可能需要 PC 宽度适配，建议做成可拉宽组件。 |
| Toast | 复用通用 Toast。 |
| 确认弹窗 | 复用通用 `ComModal`。 |

## 10. 岗位注意项

### 程序注意

- 腐败最终判定以服务端为准，客户端只展示。
- 物品移动、拖拽、拆分、全部拾取时必须校验状态，避免复制/丢失。
- 冰箱通断电、容器销毁、环境标签变化需要触发冷藏状态刷新。
- 冷藏状态下保存冻结剩余时间；断电/移出后继续腐败，不重置。
- 断线重连和服务器重启后，不能重新初始化腐败时间。
- 配置缺失时按不可腐败处理并记录日志。

### GUI注意

- 格子内不要堆太多文字，优先表达状态，再把完整说明放到详情。
- 冷藏、即将腐败、已腐败要有清晰但不夸张的视觉层级。
- 腐败压力不应做成强红色警报系统，避免玩家感到被迫上线。
- PC 宽度下的冰箱状态栏和详情信息块需要适配。

### 拼接注意

- 物品格子建议增加状态控制器：普通、可腐败、即将腐败、冷藏、已腐败、刷新中、禁用。
- 普通容器、冰箱、拾取列表的格子可共用同一状态层。
- 物品 tips 增加保质信息块，需支持显隐。
- 确认弹窗仅 P2/配置开启时启用，不要默认拼入主流程。

### 动效注意

- 冷藏生效/失效、状态刷新、Toast 使用通用轻量动效。
- 不设计远距离断电强弹窗动效。
- 即将腐败只做弱提醒，不做持续闪烁。

### 策划注意

- 需要最终确认堆叠腐败策略。
- 需要最终确认腐败产物是否允许食用。
- 需要最终确认自然寒区冷藏是暂停还是倍率延缓。
- 需要确认低频玩家保护策略是否进入 S3。
- 需要确认不可腐败食物是否在详情中显示 `不会腐败`。

## 12. 验收检查清单

- 背包、普通容器、冰箱、烹饪、拾取列表均能展示可腐败食物状态。
- 不可腐败食物不产生误导性腐败提示。
- 可腐败食物详情显示剩余保质期和腐败后产物。
- 即将腐败状态有弱提示，但不触发强弹窗。
- 通电冰箱显示冷藏生效，食物倒计时暂停或按配置倍率处理。
- 冰箱断电后界面状态刷新，并通过 Toast/界面内提示告知冷藏失效。
- 食物移出冰箱后继续腐败，移动本身不重置保质期。
- 烹饪产物按新食物配置显示完整保质期。
- 烹饪材料结算时已腐败，能给出失败提示。
- 服务端状态变化后，客户端局部刷新，不出现原食物/腐败产物显示冲突。
- 拖拽、拆分、全部拾取期间食物腐败，不出现复制、丢失或卡死。
- PC hover、移动端点击/长按、Toast、确认弹窗的双端差异明确。
- P2 腐败产物食用确认弹窗在未启用配置时不出现在主流程中。
