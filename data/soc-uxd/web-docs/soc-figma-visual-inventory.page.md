# Figma 视觉资产库

这篇资产库来自 B2 新风格迭代 GUI 效果图汇总。它更像一张“视觉资产地图”，用于快速找到系统、界面状态、移动端 / PC 效果图、商业化资源和通用模板的线索。

网页稿负责说明怎么查；完整候选资产、分组和读取口径仍以源文档为准。

## 这篇资产库解决什么问题

Figma 文件里有大量效果图、截图、分层稿和资源页。如果没有索引，设计师很难知道某个系统是否已有参考。

- 做新界面前，可以先找有没有类似视觉结构。
- 做 PC 适配前，可以先看同系统是否已有 PC 图示。
- 做商业化、商城、活动、充值、头像框等内容时，可以先查商业化资源页。
- 做 AI 深挖时，可以先判断目标是纯截图还是可读取的分层界面。

它的定位是“找线索”，不是“直接照抄”。

## 目前收录了什么

![Figma 视觉资产库](assets/soc-uxd/soc-figma-visual-inventory.svg)

| 范围 | 价值 |
|---|---|
| GUI 效果图汇总 | 覆盖局内、局外、通用、双端截图和分层稿 |
| 商业化资源 | 覆盖活动、商城、捆绑包、充值、头像框、气泡框、banner |
| 设计组件页 | 作为组件证据页，不混入界面资产计数 |
| GUI 规范相关页 | 作为规范证据页，用来补来源 |
| 空页 / 分隔页 / 封面 | 暂不作为资产深挖对象 |

源文档记录了本次读取范围、抽取口径、总量概览、重点资产组和待确认项。

## 读视觉资产时先看什么

视觉资产不是交互文档。读取时先判断它能提供哪种证据。

| 先看什么 | 为什么 |
|---|---|
| 资产类型 | 纯截图只能推断视觉，分层稿才可能读结构和文本 |
| 页面尺寸 | 判断移动端、PC、横屏、弹窗或 HUD 倾向 |
| 资产组名称 | 判断系统、业务域和页面状态 |
| 相邻状态 | 判断是否有入口、展开、确认、结果、错误等连续状态 |
| 组件痕迹 | 判断是否能回到通用组件或历史模式 |
| 待确认项 | 不把视觉推断写成已确认交互规则 |

凡是源图没有显式标注的跳转逻辑，都只能写成“待确认 / 由相邻状态推断”。

## 什么时候继续深挖

不是每张图都值得深挖。建议优先深挖下面几类。

| 深挖信号 | 深挖目标 |
|---|---|
| 新需求命中同一系统 | 抽出结构、状态矩阵和跳转逻辑 |
| PC 适配风险高 | 对比移动端与 PC 图示，找框架和输入差异 |
| 商业化页面要复用 | 查活动、商品、礼包、充值和资源展示模式 |
| 只有纯截图 | 标注证据等级，不直接生成节点结构 |
| 有分层界面 | 读取节点、文本、组件和布局关系 |

深挖后的结论应该回写到全量界面资产库或对应规范页，而不是只停留在视觉资产列表里。
