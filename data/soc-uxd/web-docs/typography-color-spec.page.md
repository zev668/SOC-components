# 字号字色规范

这篇文档说明 SOC 项目里文字样式应该怎么选：先判断文字的语义和场景，再选择字体、字号和字色。

它不是色值列表的搬运，而是一套阅读和决策方式。设计稿里不要只写“白字、灰字、橙字”，而要说明这段文字是标题、正文、次级说明、强调信息、状态提示、增益、减益，还是 HUD 特殊提示。

## 这篇规范解决什么问题

文字样式最常见的问题，是看起来差不多，但语义不一致：

- 同样是标题，有的用粗体，有的用正文体。
- 同样是次级信息，有的偏亮，有的偏暗，阅读层级不稳定。
- 强调、选中、重要提示混用同一种橙色，但没有说明语义。
- 增益和减益没有固定颜色，玩家无法快速识别正负变化。
- HUD 为了塞字越缩越小，但没有区分系统界面和 HUD 特例。

这篇规范的目标，是让文字样式从“凭感觉调”变成“按语义选”。

## 先按语义选样式

![字号字色的语义选择路径](assets/soc-uxd/typography-color-spec.svg)

选文字样式时，先问五个问题：

1. 这段文字是标题、正文、次级说明，还是强调信息？
2. 它是否表达选中、重要提示、增益、减益或在线状态？
3. 它是否属于 HUD 或 PC 持续提示这类特殊场景？
4. 字号是否低于当前场景允许的最小值？
5. 它是否放在红色按钮 / Tab 上，需要使用专用控件字色？

如果一个样式无法回答这些问题，就不要把它当成通用规则沉淀。

## 字体和字号怎么用

字体按信息重量区分：

| 使用场景 | 推荐字体 | 判断方式 |
|---|---|---|
| 标题 | 方正兰亭粗黑 | 页面或模块的主要识别信息 |
| Tab / 按钮 / 强调信息 | 方正兰亭中粗黑 | 需要快速扫读、点击或形成视觉重点 |
| 正文文本 | 方正兰亭准黑 | 常规说明、描述、提示正文 |
| 数字 / 英文 | Rust_EN_Cu | 数值、计数、英文缩写和关键数字 |

字号按场景控制下限：

| 场景 | 推荐值 | 约束 |
|---|---|---|
| 标题 | 36.48 | 可按界面需要微调 |
| 副标题 | 32.30 | 按 Figma 原文记录 |
| 正文 | 28.26 | 常规正文大小 |
| 一般系统最小字号 | 24 | 常规情况下不低于该值 |
| 极限压缩字号 | 22 | 系统界面实在摆不下才使用 |
| HUD 常规字号 | 24 | HUD 信息量较大时常用 |
| HUD 最小字号 | 20 | 仅作为 HUD 特例 |

除 HUD 外，系统界面文字不要小于 `22px`。如果为了塞内容长期依赖极限字号，优先检查布局和信息密度，而不是继续缩字。

## 字色怎么判断

基础字色按语义选择：

| 色值 | 语义 | 使用方式 |
|---|---|---|
| `#C6BFB6` | 标题 | 标题文字，强调结构识别 |
| `#CDC6BF` | 正文 | 常规正文阅读 |
| `#98938D` | 次级颜色 | 弱化说明、辅助信息 |
| `#DA622F` | 强调 / 重要提示 / 选中 | 需要玩家注意或表示当前选中 |
| `#A4BB37` | 增益 | 正向、增加、收益类信息 |
| `#F83111` | 减益 | 负向、减少、损失类信息 |
| `#38322E` | 浅底深色字 | 浅色底、按钮上的深色文字 |
| `#DDCABF` | 红色控件字色 | 仅用于红色按钮或 Tab 上的文字 |
| `#FFF7EF` | HUD / 亮色 | HUD 或高亮浅色文字 |

同一个色值可以出现在多个地方，但语义不能乱。比如橙色可以承担强调、重要提示或选中，但交互说明里要写清它在当前场景里是哪一种。

## 状态色和特殊色怎么用

特殊状态色用于状态识别，不建议随便扩展为普通装饰色。

| 色值 | 语义 |
|---|---|
| `#FFC02A` | HUD 提示黄色 |
| `#EE723E` | 游戏中 |
| `#A8A8A8` | 离线 |
| `#6FA5FD` | 大厅队伍 |
| `#A4BB37` | 在线 |
| `#A03B0A` | HUD PC 持续性提示特殊字色 |
| `#4D4A48` | 输入框默认字色 |

品质 / 皮肤名称色目前只作为样式参考。除非后续确认纳入通用语义色，否则不要把它们当成所有界面都能复用的状态色。

## 交互文档里怎么写

交互稿里涉及文字样式时，优先写语义，不要只写色值。

推荐写法：

- 标题：使用标题字体和标题字色。
- 次级说明：使用次级颜色，避免抢占主要信息。
- 价格变化：增加值用增益色，减少值用减益色。
- 红色按钮文字：使用红色控件专用字色。
- HUD 提示：允许进入 HUD 字号和 HUD 颜色规则。

如果必须临时使用新色值，要写明原因，并标记为待确认或 GUI 注意。

## 常见误区

**把颜色当装饰。**
SOC 的字色优先表达语义，不优先表达“好看”。

**把 HUD 特例带回系统界面。**
HUD 可以因为信息密度使用更小字号，但系统界面不要跟着缩。

**只写色值，不写语义。**
下游看到 `#DA622F` 时，需要知道它是强调、重要提示还是选中。

**品质色提前通用化。**
品质 / 皮肤名称色目前只是参考，纳入通用规范前不要泛用。

## 交付前快速清单

- [ ] 文字样式已按标题、正文、次级、强调等语义选择。
- [ ] 增益、减益、状态、HUD 特殊提示没有混用。
- [ ] 系统界面字号没有低于 `22px`。
- [ ] HUD 小字号只用于 HUD 场景。
- [ ] 红色按钮 / Tab 上的文字使用专用字色。
- [ ] 临时自定义字色已说明原因。
- [ ] 需要 GUI 或策划确认的文字样式已标注。
